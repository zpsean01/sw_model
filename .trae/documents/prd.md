## 1. 产品概述

ARM 固件分析结果可视化前端 — 以仪表盘形式展示流水线各阶段的分析结果（静态分析、二进制分析、建模、符号执行），帮助安全研究员和固件开发者直观理解固件结构。

- **核心目标**：将后端 API 提供的结构化数据呈现为可浏览的仪表盘界面，包括函数列表、调用图可视化、全局变量、类型定义、中断映射、状态机等
- **目标用户**：固件安全研究员、嵌入式开发者、逆向工程师
- **产品价值**：低门槛的固件分析数据可视化，无需直接阅读 JSON 即可理解分析结果

## 2. 核心功能

### 2.1 功能模块

1. **看板概览**: 所有分析类别的数量统计卡片，快速了解固件全貌
2. **静态分析**: 显示函数、全局变量、类型定义、中断、寄存器、状态机列表
3. **调用图**: 基于 NetworkX MultiDiGraph 的调用关系可视化
4. **二进制分析**: ELF 符号表函数、全局符号、DWARF 类型信息
5. **建模**: 事件架构报告、安全分析报告
6. **符号执行**: 符号执行报告、路径列表与详情

## 3. 核心流程

用户打开首页 → 看板加载所有分类统计数据 → 点击任意类别进入详情页 → 详情页从 API 加载对应数据并渲染

```mermaid
flowchart LR
  A["用户打开首页"] --> B["看板加载统计数据"]
  B --> C["点击分类卡片"]
  C --> D["API 加载对应数据"]
  D --> E["渲染详情视图"]
  E --> C
```

## 4. 用户界面设计

### 4.1 设计风格

- **主题**: 深色侧边栏 + 浅色内容区的双色布局，工业/工具风格
- **主色**: #1a1a2e（深色导航），#16213e（侧边栏），#0f3460（强调色），#e94560（高亮）
- **字体**: system-ui 原生字体栈，干净简洁
- **按钮**: 平面风格，hover 有微弱的背景变化
- **布局**: 左侧固定导航 + 右侧内容区，卡片式信息展示
- **图标**: Unicode 符号和简洁 SVG，避免外部图标库依赖

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|----------|
| 看板 | 统计卡片 | 圆角卡片网格，每张卡片显示分类名 + 数量 + 图标 |
| 详情页 | 数据表格 | 响应式表格，斑马纹交替行色 |
| 详情页 | 面包屑导航 | 当前位置指示，可回退到看板 |
| 调用图 | 可视化 | 节点-边列表展示，节点和边分段显示 |

### 4.3 响应式

桌面优先设计，表格在窄屏自动水平滚动。侧边栏在窄屏可折叠为汉堡菜单。

### 4.4 数据流

纯前端 SPA，所有数据通过 fetch API 从后端 `/api/v1/` 端点加载，运行在前端服务器（端口 5007，通过 `run_server.py` 代理 API 请求到后端 8000 端口）。无页面刷新，无服务端渲染。
